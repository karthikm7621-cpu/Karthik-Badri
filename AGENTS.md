# Autonomous Agent Interaction Guidelines

This repository permits and encourages contributions generated or assisted by Artificial Intelligence (AI) and Autonomous Agents. To maintain repository health and security, all agentic interactions must strictly adhere to the following rules:

## 1. Autonomous Contribution Rules
- **No Direct Pushes to Main:** Agents must never commit directly to the `main` branch. All agent-driven changes must be submitted via Pull Requests.
- **Verification:** Code generated by an agent must be verified through our CI/CD pipeline. The pipeline serves as the ultimate authority on code quality and test coverage.
- **Context Boundaries:** Agents must respect `.dockerignore` and `.gitignore` boundaries. They must not attempt to index, read, or modify secrets or local environment variables.

## 2. Code Attribution Requirements
- If an agent generates a substantial piece of logic or architecture, the PR description must explicitly state that the PR was AI-authored or AI-assisted.
- Agents submitting PRs should use a distinguishable commit signature if operating via a service account.

## 3. Security Limits
- Agents are strictly prohibited from modifying `.gitlab-ci.yml` or GitHub Actions workflows unless explicitly instructed by a verified human maintainer.
- Agents must not disable or circumvent any security scanning tools (e.g., Bandit, Gitleaks, MyPy).
