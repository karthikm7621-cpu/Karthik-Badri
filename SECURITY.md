# Security Policy

Security is a top priority for this project. We appreciate the efforts of the security community in helping us maintain a safe environment.

## Supported Versions

Only the latest major release branch receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| v2.x.x  | :white_check_mark: |
| v1.x.x  | :x:                |
| < 1.0   | :x:                |

## Vulnerability Reporting Process

If you discover a potential vulnerability in this project, **please do not disclose it publicly**. Instead, follow this process:

1. Send an email to `security@example.com` with the details of the vulnerability.
2. Include steps to reproduce the issue, the affected versions, and any potential mitigation strategies you have identified.
3. We will acknowledge receipt of your vulnerability report within 48 hours.

## Disclosure Policy

- We will provide regular updates on the progress of the fix.
- Once the vulnerability is resolved, we will publish a security advisory and credit the reporter (unless you prefer to remain anonymous).
- Do not publish details of the vulnerability until a fix has been released and users have had a reasonable time to patch their systems.
